# 專案交接筆記

更新日期：2026-06-27

- 目標裝置：iPhone Safari，使用者是車上乘客，不供駕駛操作。
- 遊戲核心、AI、成語庫與計分皆為本機程式；只有 Safari 語音辨識可能需要網路。
- 同音判定包含聲調。讀音已靜態放在 `idioms.js`，新增成語時必須一併加入四個字的讀音。
- 官方成語索引位於 `moe-idioms.js`，由 `scripts/import_moe_idioms.py` 從教育部 2026-03-24 版《成語典》產生；不得修改官方詞目或拼音內容。
- 四字詞語索引位於 `moe-phrases.js`，由 `scripts/import_moe_phrases.py` 從教育部 2026-03-25 版《簡編本》與《重編本》產生。只在一般辭典出現的詞目標記為 `phrase`，位置積分賽最高 1 分。
- 語音文字若不在詞庫，會以每字讀音尋找同音詞；唯一候選自動採用，多個候選時要求玩家說「確定」或「不是」。
- 玩家答案不再要求收錄於詞庫；任何四個中文字都可建立 `source: free` 的四字詞語並繼續遊戲，位置積分最高 1 分。詞庫只負責 AI、音讀與語音錯字修正。
- 開局方式可選系統從至少有三個同字接續詞的成語中隨機出題，或由玩家自由說第一個詞；玩家出題不計分並換下一位。
- 系統隨機開局先等機率抽「尾字群組」，再抽該群組中的成語，避免「說、功」等高頻尾字反覆出現。
- 積分模式接受成語庫中未使用過的成語；沒有命中目標字／音時得 0 分，但仍以該成語尾字繼續下一回合。
- AI：輕鬆模式偏向留較多出口；標準模式先取最高分；挑戰模式先取最高分，再偏向留下較少出口。
- `navigator.onLine` 只作初步訊號；實際連線以未快取的 `online-check.txt` 確認。Service Worker 不得快取該檔案。
- Service Worker 對遊戲資源採「有網路先更新、失敗再讀快取」，避免部署後仍長期使用舊成語庫。
- Windows 端可驗證網頁流程，但真正的麥克風權限與離線語音辨識仍須在實體 iPhone Safari 測試。
- 使用者已在 2026-06-27 同意建立公開 GitHub Pages 網站。
- 多人模式可語音說「放棄」；必須再說「確定」才以 0 分換人，說「取消」則維持原回合。放棄不改變目前目標字。
